# Missing Trixie

Use the CloudNativePG `ClusterImageCatalog` resources generated by this repository. Apply `cloudnative-pg-timescaledb/catalog/catalog-standard-bookworm.yaml` as the secondary Debian variant.

The generated resource names include `cloudnative-pg-timescaledb-standard-bookworm`.

```yaml
apiVersion: postgresql.cnpg.io/v1
kind: Cluster
metadata:
  name: app-db
spec:
  imageCatalogRef:
    apiGroup: postgresql.cnpg.io
    kind: ClusterImageCatalog
    name: cloudnative-pg-timescaledb-standard-bookworm
    major: 18
```

PostgreSQL `17` maps to catalog `major: 17`; PostgreSQL `18` maps to catalog `major: 18`.

Catalog entries prefer published multi-platform index digest references:

```yaml
image: ghcr.io/pnetcloud/cloudnative-pg-timescaledb:18-pg18.4-ts2.27.2-20260609-bookworm@sha256:bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb
```

Catalogs are generated from release-complete published images. Catalogs must not reference unpublished images, unsigned digests, missing digests, wrong PostgreSQL majors, or wrong Debian variants. Do not use `latest` as the primary CloudNativePG catalog path. Standard catalogs omit PostgreSQL `19beta1`; any PG19 beta catalog example must be explicitly experimental.
